1. **Audit log date formatting** (`packages/crud/src/components/CRUDTable.tsx`)
   In the date renderer, change `value.toLocaleDateString()` to:
   ```ts
   new Date(value as string).toLocaleDateString(undefined, { dateStyle: 'medium', timeStyle: 'short' })
   ```
   Coerces string to Date before formatting. Adds locale options for readable output.

2. **Page `<title>` per resource** (`apps/api/src/app/admin/[resource]/page.tsx`)
   ```ts
   export async function generateMetadata({ params }: { params: { resource: string } }) {
     const config = CRUDConfigs[params.resource as keyof typeof CRUDConfigs];
     return { title: config ? `${config.label} | Admin` : 'Admin' };
   }
   ```

### Expansion Items (after DX polish pass)

5. **Pagination** (`packages/crud/src/router-factory.ts` + `CRUDTable.tsx` + `CRUDResourceClient.tsx`)
   - Add `page?: number` param to tRPC list input (default: 1). Default page size: 50.
   - `router-factory`: calculate `skip = (page - 1) * 50`, pass `take: 50, skip` to Prisma.
   - Return `{ items, total }` from list procedure (count query needed for Prev/Next state).
   - `CRUDTable`: add Next/Prev buttons, show "Showing X-Y of N". Stateless — controlled by parent.
   - `CRUDResourceClient`: manage page state (`useState(1)`), pass to tRPC query as `{ page }`.
   - **Backward compat**: `page` is optional, defaults to 1. No existing call site changes needed.
   - **Performance note**: `{ items, total }` requires two queries (findMany + count). Acceptable for
     admin panels at typical client scale (< 100k records). If count becomes slow, defer to TODOS.md
     as "pagination count optimization" — the fix is a cached materialized count, not a schema change.

6. **Toast notifications** (`apps/api/src/app/admin/[resource]/CRUDResourceClient.tsx`)
   Toast calls live in `CRUDResourceClient.tsx` mutation callbacks (where `mutateAsync` is called),
   not in `CRUDPage.tsx`. Import `useToast` from `@repo/ui`:
   ```ts
   const { toast } = useToast();
   // In each mutation success:
   onCreate: async (data) => {
     try {
       await createMutation.mutateAsync({ data });
       toast({ title: `${config.label} created` });
       // ...close modal, refetch
     } catch (err) {
       toast({ title: `Error: ${(err as Error).message}`, variant: 'destructive' });
     }
   }
   // Same pattern for onUpdate ("Updated"), onDelete ("Deleted")
   // Error toast uses destructive variant for red/error styling
   ```

7. **`pnpm dev:reset`** (`package.json` root scripts)
   ```json
   "dev:reset": "pnpm --filter db exec prisma db push --force-reset && pnpm --filter db exec prisma db seed"
   ```
   Add note to README Quick Start: "Wipe and re-seed: `pnpm dev:reset`"

8. **Column sorting** (`packages/crud/src/router-factory.ts` + `CRUDTable.tsx`)
   - Add `orderBy?: { field: string; direction: 'asc' | 'desc' }` to tRPC list input.
   - **Field validation (security)**: before applying sort, validate `input.orderBy.field` against
     the config's declared field names: `if (!config.fields.some(f => f.name === field)) throw new TRPCError(...)`.
     This prevents sorting by unlisted/private columns (e.g., `password`, `hashedToken`).
   - In list handler: `prisma[model].findMany({ ..., orderBy: input.orderBy ? { [input.orderBy.field]: input.orderBy.direction } : undefined })`.
   - `CRUDTable`: wire TanStack Table `onSortingChange` to call `setOrderBy` + `setPage(1)` in CRUDResourceClient.
     Sort change always resets to page 1.
   - `CRUDResourceClient`: manage `orderBy` state alongside `page` state, thread both into tRPC query input.
   - Default: no sort applied (DB insertion order, same as today). Zero behavioral change until user clicks a header.

9. **`pnpm crud:new`** (`scripts/crud-new.mjs` — already exists as stub)
   Interactive readline flow:
   1. Prompt: "Model name (e.g. post):"
   2. Prompt loop:
      ```
      Add field (format: name type required — e.g. "title text true" or "price number false")
      Valid types: text, number, boolean, textarea, email, url, date, select
      Press Enter with no input to finish adding fields.
      ```
   3. Build `defineCRUD` config from collected fields
   4. Write `apps/api/src/crud/<model>.ts`
   5. Call `pnpm crud:scaffold <model>` internally (spawns subprocess)
   Output: same ✓/⚠ feedback as crud-scaffold.ts.
   File: `scripts/crud-new.mjs` (check if stub already has structure, extend if so).
   **Validation**: required must be 'true' or 'false'; type must be in [text, number, boolean, textarea, email, url, date, select]. Reprompt on invalid input with error message.
   **Note on `select` type**: if user enters `select`, prompt for options: "Options (comma-separated values, e.g. draft,published,archived):". Build `options: [{value, label}]` from the comma-separated list.
   **Note on "The Assignment" timing**: Run "The Assignment" (item 10) AFTER all items ship; it does not block the PR merge. Update README with measured TTHW post-verification.
